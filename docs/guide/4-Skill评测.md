# Skill 评测

> **版本**: v5.0.0
> **更新日期**: 2026年4月

---

## 1. 评测流程

传统的 Skill 评测大多只关注"任务是否完成"，但这远远不够——即使结果正确，Agent 可能跳过了关键的备份步骤，也可能 Token 消耗过高导致成本不可接受。Skill-insight 的评测不只看结果，还追踪执行过程，提供"结果 + 路径 + 成本"的综合分析。

整体流程如下：

```
用户任务 → Agent 执行 → 数据采集 → 链路追踪 → 多维评测 → 归因分析
```

平台会自动采集 Agent 的执行轨迹和会话记录，基于评估模型进行归因分析，并将结果直观地呈现在看板上。

## 2. 评测指标

### 2.1 效果指标

| 指标 | 定义 | 说明 |
|:---|:---|:---|
| **准确率** | 各评分项得分的加权平均值 | [0,1]之间的数值，衡量执行结果与预期的匹配程度。 |
| **Skill 召回率** | 实际使用的 Skill 中属于期望 Skill 的比例 | 衡量 Skill 的实际命中情况 |
| **Skill 提升率** | 使用 Skill 前后准确率的相对提升 | 衡量 Skill 对任务完成质量的实际贡献 |

### 2.2 效率指标

| 指标 | 定义 | 说明 |
|:---|:---|:---|
| **时延** | 任务开始到结束的耗时 | 表示任务的响应速度 |
| **模型调用次数** | 完成任务调用模型的总次数 | 用于判断执行策略是否冗余 |
| **工具调用次数** | 完成任务调用工具的总次数 | 用于判断执行策略是否冗余 |

### 2.3 成本指标

| 指标 | 定义 | 说明 |
|:---|:---|:---|
| **Token 总量** | 输入 Token + 输出 Token | 用于成本核算 |
| **模型费用** | Token 数量 × 模型单价 | 估算的 API 调用费用 |
| **CPSR** | 总费用 / 成功执行次数 | 获得一个正确答案的平均成本（准确率 > 0.8 视为成功） |
| **上下文利用率** | 单次最大 Token 数 / 模型上下文窗口上限 × 100% | 超过 90% 会标红告警，提示有截断风险 |
| **输入/输出 Token** | 分别统计输入和输出的 Token 数 | 辅助分析 Token 消耗构成 |

### 2.4 稳定性指标

| 指标 | 定义 | 说明 |
|:---|:---|:---|
| **工具调用错误率** | 工具调用错误次数 / 总调用次数 | 反映工具定义或模型推理能力是否存在不足 |

## 3. 配置评测数据集

评测数据集用于定义评测基准，包含问题和对应的标准答案。

### 3.1 手动输入

1. 进入看板的 **数据集管理** 标签页
2. 点击 **+ 新增配置**
3. 填写问题（Query）
4. 选择"手动输入"模式，填写标准答案
5. 保存

### 3.2 上传文档

1. 进入 **数据集管理** 标签页
2. 点击 **+ 新增配置**
3. 填写问题（Query）
4. 选择"上传文档"模式，上传案例文档（支持 .md、.txt、.pdf）
5. 系统会自动解析文档并提取评分项，等待解析完成后保存

解析状态说明：🔄 解析中、✅ 完成、❌ 失败。

### 3.3 评分项自动提取

系统会从标准答案中自动提取以下两类评分项：

- **Root Causes**（根因分析要点）：识别关键结论和诊断结果
- **Key Actions**（关键操作步骤）：识别执行的操作序列

## 4. 查看评测结果

### 4.1 主页概览

主页面提供全景概览，包括：

- **概览卡片**：展示各任务的核心指标汇总。准确率大于 80% 显示为绿色，否则为黄色
- **对比分析图表**：包含平均时延对比、平均 Token 消耗对比、平均准确率对比三个图表，可直观对比不同框架或模型的表现
- **执行记录表格**：列出每次执行的时间、框架、问题、时延、Token、准确率、模型、标签等信息

### 4.2 执行记录详情

点击任意执行记录的"详情"按钮，可以查看：

**评测结果摘要**：展示状态（PASS/FAIL）、准确率、时延、Token 等关键数据。准确率 > 0.8 判定为 PASS，否则为 FAIL。

**判题理由**：逐项展示每个评分项的匹配程度（0-100%）、扣分原因说明，以及最终分数的计算过程。

**Skill 分析**：识别哪些扣分项可以通过优化 Skill 来改善。平台会区分以下几种情况：

| 情况 | 是否 Skill 问题 | 说明 |
|:---|:---|:---|
| Skill 中缺少相关信息 | 是 | 需要补充 Skill 内容 |
| Skill 有信息但 Agent 未使用 | 否 | 属于模型能力问题 |
| 评分项要求通用知识 | 否 | 不在 Skill 职责范围内 |
| 运行时环境问题 | 否 | 网络、权限等外部因素 |

**执行流程分析**：平台会绘制实际执行的 Mermaid 流程图，并支持与 Skill 的预期流程进行对比。对比结果用以下标记区分：

- ✅ 符合预期：实际执行符合 Skill 规定的步骤
- ⚠️ 部分偏离：与 Skill 规定存在偏差
- ❌ 非预期调用：执行了 Skill 中未规定的操作
- ⭕ 跳过：Skill 中规定了但实际未执行的步骤

点击问题步骤序号可以查看对应的对话数据。

## 5. 分析评测趋势

### 5.1 趋势分析

看板提供时延、Token 消耗、准确率的趋势图，帮助你：

- 观察指标随时间的变化情况
- 识别异常峰值和周期性波动
- 评估 Skill 优化后的实际效果

### 5.2 对比分析

支持以下维度的对比：

- **框架对比**：不同 Agent 框架在准确率、时延、Token 消耗上的表现差异
- **模型对比**：同一框架下，不同底层模型的效果和成本对比
- **标签对比**：不同类型任务的表现差异，帮助识别薄弱环节和 Skill 覆盖盲区

## 6. 导出评测结果

1. 进入执行记录详情页
2. 点击右上角"📤 导出"按钮
3. 选择保存位置

导出的 HTML 文件可以离线查看和分享。
