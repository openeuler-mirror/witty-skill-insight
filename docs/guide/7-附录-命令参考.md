# 附录 - 命令参考

> **版本**: v5.0.0
> **更新日期**: 2026年4月

---

## 1. 平台服务管理

| 命令 | 说明 |
|:---|:---|
| `npx @witty-ai/skill-insight install` | 一键安装平台及所有组件 |
| `npx @witty-ai/skill-insight start` | 启动服务（默认 3000 端口） |
| `npx @witty-ai/skill-insight start --port <端口>` | 指定端口启动 |
| `npx @witty-ai/skill-insight stop --port <端口>` | 停止指定端口的服务 |
| `npx @witty-ai/skill-insight restart` | 重启服务 |
| `npx @witty-ai/skill-insight status` | 查看服务运行状态 |
| `npx @witty-ai/skill-insight logs` | 查看服务日志 |

## 2. 数据采集通道

### 安装与配置

| 命令 | 说明 |
|:---|:---|
| `curl -sSf http://<IP>:<PORT>/api/setup \| bash` | Linux/macOS 一键配置数据采集 |
| `irm http://<IP>:<PORT>/api/setup \| iex` | Windows 一键配置数据采集 |

### Watcher 管理

| 命令 | 说明 |
|:---|:---|
| `bash ~/.witty/start_watchers.sh` | 启动所有数据上报通道 |
| `bash ~/.witty/stop_watchers.sh` | 关闭所有数据上报通道 |
| `bash ~/.witty/start_claude_watcher.sh` | 单独启动 Claude Code 上报通道 |
| `bash ~/.witty/start_openclaw_watcher.sh` | 单独启动 OpenClaw 上报通道 |
| `rm ~/.opencode/plugins/Skill-Insight.ts` | 关闭 OpenCode 上报通道 |

### 日志查看

| 命令 | 说明 |
|:---|:---|
| `tail -f ~/.witty/logs/claude_watcher.log` | 查看 Claude Code 上报日志 |
| `tail -f ~/.witty/logs/openclaw_watcher.log` | 查看 OpenClaw 上报日志 |

## 3. Skill 生成与优化

### Agent 能力注入

| 命令 | 说明 |
|:---|:---|
| `npx skills add https://gitcode.com/openeuler/skill-insight.git` | 给 Agent 注入 Skill 生成/优化能力 |

### 本地 CLI - Skill 生成器

| 命令 | 说明 |
|:---|:---|
| `python skill_gen_cli.py --input <输入> --output <输出目录>` | 根据文档生成 Skill |

入口脚本路径：`skills/skill-generator/scripts/skill-gen/skill_gen_cli.py`

### 本地 CLI - Skill 优化器

| 命令 | 说明 |
|:---|:---|
| `python scripts/main.py --mode static --input <Skill目录>` | 静态检查与规范化 |
| `python scripts/main.py --mode warm --input <Skill目录>` | 结合日志的针对性修复 |
| `python scripts/main.py --mode hybrid --input <Skill目录>` | 静态检查 + 日志分析综合优化 |

入口脚本路径：`skills/skill-optimizer/scripts/main.py`

## 6. 排障常用

| 命令 | 说明 |
|:---|:---|
| `lsof -i :<端口>` | 查找占用指定端口的进程 |
| `kill -9 <PID>` | 强制终止进程 |
| `cat ~/.witty/.env` | 查看采集配置文件 |
| `grep WITTY_INSIGHT_API_KEY ~/.witty/.env` | 检查 API Key 配置 |
| `ps aux \| grep watcher` | 检查 Watcher 进程状态 |
