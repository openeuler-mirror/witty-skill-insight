# Skill 生成

> **版本**: v5.0.0
> **更新日期**: 2026年4月

---

## 1. 功能说明

Skill-insight 的生成功能不是简单地将文档转换为 Skill，而是采用"去冗余、合相似、抽模式"的方法，从大量相似的案例文档中提炼出标准化、可复用的 Skill。

具体来说：

- **去冗余**：从案例文档中剔除重复描述和噪声信息，保留核心特征、前置检查与关键操作步骤
- **合相似**：基于文本聚类和语义理解，将同类问题的多份文档合并，提炼共性逻辑
- **抽模式**：在合并基础上，结合场景规范和操作标准，提炼通用的执行路径，生成可复用的 Skill

这样做的好处是：用更少的 Skill 覆盖更多场景，在提升召回率的同时降低 Token 消耗。

生成过程通过在 Agent 终端中使用自然语言触发

---

## 2. 前置准备

### 2.1 给 Agent 注入生成能力

在 Agent 的工作终端中执行以下命令，安装 Skill-insight Meta-Skill：

```bash
npx skills add https://gitcode.com/openeuler/witty-skill-insight.git
```

根据指引，选择安装skill-generator，并选择合适的集成框架（如Opencode）。完成后，Agent 就可以通过自然语言调用 Skill 生成能力。

### 2.2 支持的输入类型

你可以使用以下任意一种作为输入：

- Markdown 或 TXT 文档
- PDF 文档
- 一个目录（包含多份文档）
- 一个可访问的 URL

建议输入内容尽量"任务导向"，比如包含：背景说明、现象描述、定位步骤、关键命令、预期输出、回滚方案等。内容越具体，生成的 Skill 质量越高。

## 3. 在 Agent 终端中生成 Skill

### 3.1 基本用法

在 Agent 终端中输入类似以下内容即可触发生成：

> 根据案例文档：`<本地路径或 URL>` 生成一个 Skill。

### 3.2 进阶用法

如果希望生成的 Skill 更加规范和可复用，可以在指令中补充更多要求：

> 根据案例文档：`<路径或 URL>` 生成一个 Skill。
> 要求：把硬编码参数做成入参；给出前置检查；输出要包含关键结论与证据。

### 3.3 批量生成与去重

当你有多份相近的案例文档时，可以一次性输入，系统会自动合并相似内容、去除冗余：

> 根据这些案例文档：`<目录或文件列表>` 生成 Skill，并尽量合并相似内容，避免重复 Skill。

这是 Skill-insight 生成能力的核心优势——不是逐文档生成，而是跨文档聚合提炼。

### 3.4 生成后自动验证

建议生成完成后立即做一次自测：

> 生成 Skill 后，用一个小例子自测一下：以"<你关心的场景>"为输入，跑一遍并展示结果。

---

## 4. 确认 Skill 已生效

生成完成后，可以通过以下方式确认 Skill 是否正常工作：

- **直接验证**：让 Agent 用新生成的 Skill 完成一次你熟悉的任务，看结果是否符合预期
- **检查文件**：在 Skill 存放目录中查看是否生成了新的 Skill 包（必须包含 `SKILL.md` 文件）
- **看板查看**：如果已配置数据采集，生成和执行轨迹会自动上报到看板，可在执行历史中查看
